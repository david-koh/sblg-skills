# sblg-learn — 새로운 분야 학습

전문가 입력 → 제1원리 분해 → 교차 학습 → 전이 학습 파이프라인으로 새로운 분야의 근본 원리를 세우고, 대화형 학습 세션으로 지속 학습하는 스킬.

로드맵에서 끝나지 않는다. 실제 학습까지 이어진다.

## Commands

| 커맨드 | 설명 |
|--------|------|
| `/sblg-learn` | 새 학습 시작 (Phase 1부터) |
| `study {토픽}` | 특정 토픽 학습 세션 |
| `study next` | 다음 미완료 토픽 학습 |
| `quiz` | 현재 토픽 자가 점검 |
| `status` | 전체 진도 요약 |
| `continue` | 이전 대화에서 이어서 (\_status.md 기반) |

## Session Continuity

새 대화에서 `continue` 또는 `status` 입력 시:
1. `~/.sblg/learn/` 디렉토리에서 가장 최근 세션 탐색
2. `_status.md` 읽어서 현재 Phase와 진도 파악
3. 남은 작업부터 이어서 진행

## Web Search

검색이 필요한 단계에서 사용 가능한 웹 검색 도구를 사용한다.
검색 도구가 없으면 "검색 불가 — 내부 지식으로 진행" 안내 후 계속.

## Workflow

### Phase 1: 전문가 패널 소환 (입력)

사용자가 배우고 싶은 분야를 입력하면 해당 분야의 Top 0.1% 실무 전문가 페르소나 3~5명을 자동 생성한다. 각 전문가는 단순 지식 나열이 아니라 실무 엣지 케이스와 초보자의 흔한 오해를 짚어야 한다.

**페르소나 생성 기준:**
- 같은 분야 2~3명 (서로 다른 세부 전문성)
- 인접 분야 1~2명 (다른 시각 제공)
- 각 페르소나에 이름, 전문 영역, 경력 배경 부여

**병렬 subagent로 각 전문가가 독립적으로 답변:**
1. "이 분야의 줄기(trunk)와 큰 가지(big branches)는 무엇인가?"
2. "초보자가 흔히 하는 잘못된 가정 3가지는?"
3. "2주 안에 핵심을 잡으려면 어떤 순서로 배워야 하는가?"
4. "실무에서 가장 자주 부딪히는 엣지 케이스는?"

각 전문가 답변 완료 후 사용자에게 요약 보고: "Phase 1 완료: 전문가 N명 의견 수집. [주요 공통점/차이점 1줄 요약]"

산출물: `expert-opinions.md`

### Phase 2: 퍼스트 프린시플 분해 (원자 단위 해체)

Phase 1의 전문가 의견은 입력 재료일 뿐이다. 이 단계의 본질은 분야를 둘러싼 복잡한 용어, 업계 상식, 관행을 모두 지우고, 더 이상 쪼갤 수 없는 근본 원리(원자)만 남기는 것이다.

**Step 1: 변수/상수 분리**

전문가들이 나열한 개념들에서:
- **상수** (타협 불가능한 본질): 물리법칙, 수학적 제약, 논리적 필연. 이것이 없으면 이 분야가 존재할 수 없다.
- **변수** (일반적 착각/관행): "보통 이렇게 배운다", "이 순서로 해야 한다", "이 도구를 써야 한다"

**Step 2: 변수 제거 (소크라테스식 심문)**

각 변수에 대해:
1. "이 가정의 출처가 뭐지?" — 누가 처음 말했나, 근거가 뭔가
2. "이게 물리적 제약인가, 관행인가?" — 제거하면 세상이 무너지나?
3. "이 가정 없이 성공한 사례가 있나?" — WebSearch로 반증 탐색
4. "이 가정이 틀리면 어떤 가능성이 열리나?"

제거 가능한 변수는 지우고, 진짜 상수만 남긴다.

**Step 3: 원자 단위 도출**

남은 상수들이 "더 이상 쪼갤 수 없는 원자"인지 검증. 쪼갤 수 있으면 한 번 더 분해. 최종: 3~7개의 근본 원리.

**Step 4: 인과관계 재구성**

원자들을 인과관계로 연결: "A이므로 B, B이므로 C, 따라서 이 분야는 이런 구조다." 이 분야가 왜 현재 형태로 존재하는지 논리적으로 전개.

완료 후 사용자에게 보고: "Phase 2 완료: 근본 원리 N개 도출. [원자 목록 1줄]. 변수 M개 제거."

산출물: `fundamentals.md` (상수/변수 분류표 + 원자 목록 + 인과관계 전개)

### Phase 3: 교차 학습 (연결)

단순히 "비슷한 거 찾기"가 아니다. 사용자가 이미 깊이 아는 분야(A)의 근본 원리를 추출하고, 새 분야(B)의 원자(Phase 2 산출물)와 구조적으로 1:1 매핑한다. 비유가 아니라 같은 원리가 다른 옷을 입고 있는 것.

**Step 1: 사용자의 기존 전문 분야 파악**

"당신이 가장 깊이 아는 분야는 무엇인가?" — 이미 알고 있으면 스킵.

**Step 2: 기존 분야(A)의 근본 원리 추출**

Phase 2와 동일한 원자 분해를 A에도 적용. (간략하게 — A는 사용자가 이미 아니까)

**Step 3: 구조적 유사성 매핑**

A의 원자와 B의 원자를 1:1 대응:

| A의 원리 | B의 원리 | 구조적 유사성 근거 |
|----------|----------|-------------------|
| ... | ... | ... |

"비유"가 아니라 "같은 원리의 다른 표현"인지 검증.

**Step 4: 매핑 검증**

WebSearch는 여기서 사용 — 연결을 "발견"하는 게 아니라 "검증"하는 데. "이 구조적 유사성이 실제로 성립하는 사례가 있나?"

완료 후 보고: "Phase 3 완료: 기존 분야 '{A}'와 N개 원리 매핑 완료."

산출물: `connections.md`

### Phase 4: 전이 학습 → 학습 로드맵

Phase 3의 매핑을 바탕으로 A의 학습 패턴을 B에 재구성. 구조적 유사성을 활용해 B의 원자를 효율적으로 체득할 수 있는 학습 경로를 설계.

**토픽별 학습 계획 생성:**
- 각 토픽: 이름, 예상 시간, 선행 조건, 자가 점검 퀴즈 3~5개
- 토픽 순서: 원자(줄기) → 인과관계(가지) → 응용(잎) 순서
- 교차 학습 연결점 표시: "이 토픽은 A의 {원리}와 같은 구조"

완료 후 보고: "Phase 4 완료: 토픽 N개 로드맵 생성. 예상 총 학습 시간: ~X시간."

산출물: `roadmap.md`
상태 업데이트: `progress.md` (토픽별 상태: pending)

### Phase 5: 학습 세션 (반복)

`study {토픽}` 또는 `study next`로 토픽 학습을 시작한다. 로드맵에서 끝나지 않고 여기서 실제로 배운다.

**각 토픽 학습 흐름:**

1. **청크 단위 설명** — 한 번에 쏟아내지 않는다. 핵심을 깊게 찌르되 소화 가능한 분량만.
2. **역질문** — 설명 후 사용자에게 마이크를 넘긴다: "이걸 비개발자에게 설명한다면?", "이 원리를 당신의 프로젝트에 적용하면?"
3. **현미경 피드백** — 사용자 답변에 무조건 칭찬하지 않는다. 논리적 비약, 오개념, 실무 맹점을 날카롭지만 건설적으로 교정.
4. **교차 학습 연결** — "기존에 알던 {A의 원리}와 비슷하다" 연결점 제시.
5. **30초 요약** — 면접/회의에서 바로 쓸 수 있는 한 문단.
6. **자가 점검 퀴즈** 3~5개 → 사용자 응답 → 채점 + 피드백.
7. **심화 압박** — 퀴즈 통과 시 시니어 면접 수준 딜레마로 깊이 확장.

퀴즈 결과에 따라 progress.md 업데이트: passed / weak

산출물: `studies/topic-{번호}-{이름}.md`

### Phase 6: 약점 분석 + 반복

progress.md 기반으로 weak 토픽 재학습 제안. `status`로 전체 진도 확인. 모든 토픽 passed 시 완료.

## Output Structure

```
~/.sblg/learn/
└── {주제}-{YYYYMMDD}/
    ├── _status.md
    ├── progress.md
    ├── expert-opinions.md
    ├── fundamentals.md
    ├── connections.md
    ├── roadmap.md
    └── studies/
        ├── topic-01-{name}.md
        └── ...
```

## _status.md Format

```markdown
# Session Status
- subject: {주제}
- current_phase: {1~6}
- started_at: {날짜}
- last_updated: {날짜}
- expert_count: {N}
- fundamentals_count: {N}
- topics_total: {N}
- topics_passed: {N}
- topics_weak: {N}
- next_action: {다음에 해야 할 구체적 작업}
```
